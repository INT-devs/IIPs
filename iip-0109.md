# IIP-109: Duplicate Transaction Output Prevention

## Abstract

This IIP documents INTcoin's adoption of BIP-30, preventing blocks
from containing transactions whose outputs already exist as unspent
UTXOs. The mechanism is implemented without modification at the
consensus level and is additionally enforced at mempool acceptance.

## Specification

Implements [BIP-30](https://github.com/bitcoin/bips/blob/master/bip-0030.mediawiki)
without divergence at the consensus layer.

A block is invalid if it contains any transaction whose txid matches
a transaction that already has unspent outputs in the UTXO set. This
prevents overwriting existing UTXOs.

### Mempool enforcement

In addition to consensus-level validation, the mempool rejects
transactions whose outputs already exist in the UTXO set. For each
output index N in the submitted transaction, if
`COutPoint(txid, N)` already exists as an unspent UTXO, the
transaction is rejected with error `"txn-already-known"`.

This prevents duplicate transactions from being relayed across the
network and gives the submitter an immediate error.

### Relationship to IIP-0105

IIP-0105 (BIP-34, height in coinbase) makes BIP-30 violations
practically impossible for coinbase transactions because the height
prefix ensures unique txids. IIP-0109 remains necessary as a
defence-in-depth check for non-coinbase transactions.
