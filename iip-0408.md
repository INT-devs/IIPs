# IIP-408: Output Descriptors
## Abstract
Documents INTcoin's adoption of BIP-380 output descriptors for expressing output scripts in a human-readable, composable format. Adapted for Dilithium5 key types and IIP-0301 HD derivation paths.
## Specification
Implements [BIP-380](https://github.com/bitcoin/bips/blob/master/bip-0380.mediawiki) with the following divergences:
- **Key expressions** use 2,592-byte Dilithium5 public keys instead of 33-byte compressed secp256k1 keys.
- **HD path expressions** use IIP-0301 derivation (SHA3-256, coin type 2210) instead of BIP-32.
- **Extended key prefixes** are `ixpub`/`ixprv` instead of `xpub`/`xprv`.
- **Checksum** uses SHA3-256 instead of SHA-256.
### Supported descriptor types
- `pk(KEY)` — Pay to public key
- `pkh(KEY)` — Pay to public key hash (standard P2PKH with OP_CHECKSIG_DILITHIUM)
- `multi(k,KEY1,KEY2,...)` — k-of-n via PST workflow (no on-chain multisig opcode)
- `sortedmulti(k,KEY1,KEY2,...)` — sorted variant
- `sh(SCRIPT)` — Pay to script hash
