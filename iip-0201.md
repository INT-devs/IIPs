# IIP-201: Compact Block Relay

## Abstract

This IIP documents INTcoin's adoption of BIP-152 compact block relay.
Compact blocks reduce block propagation bandwidth by transmitting
short transaction IDs instead of full transactions, relying on the
receiver's mempool to reconstruct the block. The mechanism is
implemented without modification.

## Specification

Implements [BIP-152](https://github.com/bitcoin/bips/blob/master/bip-0152.mediawiki)
without divergence.

### Messages

| Message | Direction | Description |
|---------|-----------|-------------|
| SENDCMPCT | Both | Signal compact block support and mode |
| CMPCTBLOCK | Sender → Receiver | Compact block with short tx IDs |
| GETBLOCKTXN | Receiver → Sender | Request missing transactions |
| BLOCKTXN | Sender → Receiver | Missing transactions response |

### Modes

- **Low-bandwidth mode:** Receiver gets a compact block only after
  requesting it via a header or inv announcement.
- **High-bandwidth mode:** Sender pushes compact blocks immediately
  on new tip, before the receiver requests them.

### Short transaction IDs

6-byte short IDs computed from a SipHash of the transaction hash,
keyed per-block with a random nonce.
