# IIP-212: Tor Hidden Service Transport

## Abstract

This IIP specifies how INTcoin nodes operate as Tor v3 hidden
services and connect to other nodes over the Tor network. Tor
integration provides IP address privacy, censorship resistance,
and NAT traversal for nodes behind restrictive firewalls.

## Motivation

Clearnet P2P connections reveal the node operator's IP address to
peers and network observers. Running as a Tor hidden service hides
the node's IP while remaining reachable by other Tor-connected
peers. This is particularly important for INTcoin's post-quantum
security model — metadata privacy should complement cryptographic
privacy.

## Specification

### Tor Control Protocol

INTcoin connects to the local Tor daemon via the control port
(default 9051) using SAFECOOKIE, COOKIE, or HASHEDPASSWORD
authentication (in priority order). On successful authentication
the node creates an ephemeral v3 onion service mapped to its P2P
listening port.

### Onion Address Format

Tor v3 addresses are 56 characters (ed25519 public key + checksum
+ version), encoded as `<address>.onion`. INTcoin stores these as
32-byte binary addresses internally, serialised via ADDRv2
(IIP-0202) with network ID `0x03` (TORV3).

### Connection Flow

1. Node starts and detects Tor availability via control port probe
2. Authenticates using best available method
3. Creates ephemeral hidden service: `ADD_ONION NEW:ED25519-V3 Port=<p2p_port>`
4. Advertises onion address to peers via ADDRv2 messages
5. Outbound Tor connections routed through SOCKS5 proxy (default 127.0.0.1:9050)

### SOCKS5 Integration

All outbound connections to `.onion` addresses are routed through the
configured SOCKS5 proxy per RFC 1928/1929. Stream isolation is
supported — each connection uses a unique SOCKS5 authentication
credential to request a separate Tor circuit.

### Configuration

```ini
# intcoin.conf
tor=1                           # Enable Tor support
torcontrol=127.0.0.1:9051      # Tor control port
torpassword=<password>          # Optional: hashed password auth
onlynet=onion                   # Optional: Tor-only mode
proxy=127.0.0.1:9050            # SOCKS5 proxy for outbound
```

### Network Ports

| Network | Tor P2P Port |
|---------|-------------|
| Mainnet | 2210 |
| Testnet | 12210 |
| Signet  | 22210 |

### Privacy Considerations

- Tor-only mode (`onlynet=onion`) prevents any clearnet connections
- Stream isolation prevents circuit correlation between connections
- Ephemeral keys mean the onion address changes on restart unless
  persistent keys are configured
- Tor transport is always wrapped in IIP-0200 V2 encryption, providing
  post-quantum confidentiality on top of Tor's circuit encryption

## Compatibility

Tor v3 addresses require ADDRv2 (IIP-0202) for relay. Nodes that
do not support ADDRv2 cannot learn about or connect to Tor-only peers.
