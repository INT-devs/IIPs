# IIP-418: Decentralized Exchange Protocol

## Abstract

This IIP specifies a decentralised exchange (DEX) protocol for
INTcoin, enabling trustless on-chain order matching and settlement
for INT trading pairs.

## Motivation

Centralised exchanges are custodial — users deposit funds and trust
the exchange to safeguard them. A DEX protocol enables peer-to-peer
trading directly on-chain using atomic swaps (IIP-0113) or
HTLC-based order matching.

## Specification

### Order Types

| Type | Description |
|------|-------------|
| Limit | Execute at specified price or better |
| Market | Execute immediately at best available price |

### Order Book

Orders are published via a P2P gossip protocol (similar to
Lightning gossip). Each order contains:
- Trading pair (e.g., INT/BTC)
- Side (buy/sell)
- Amount and price
- Expiry time
- Maker's HTLC parameters

### Settlement

Orders are settled via atomic swaps (IIP-0113):
1. Taker selects an order from the order book
2. Taker initiates HTLC on their chain
3. Maker responds with matching HTLC
4. Atomic swap completes the trade

### On-Chain Order Matching (Future)

For INT/INT trading pairs (e.g., token-like constructs):
- Orders encoded as special transactions
- Matching engine as a covenant (IIP-0114)
- Settlement in a single atomic transaction

## Status

Design phase. Depends on atomic swap protocol (IIP-0113) maturity.

## Reference Implementation

Planned for Q2 2029.
