# IIP-215: Dandelion++ Transaction Relay

## Abstract

This IIP specifies Dandelion++ transaction relay for INTcoin,
providing origin privacy by splitting relay into a "stem" phase
(linear propagation) and a "fluff" phase (normal diffusion).

## Motivation

Standard transaction relay broadcasts to all peers simultaneously,
allowing network observers to identify the originating node via
timing analysis. Dandelion++ makes it significantly harder to trace
transactions back to their source.

## Specification

### Two-Phase Relay

1. **Stem phase:** Transaction is forwarded to a single randomly
   selected peer. Each stem node independently decides (with
   probability ~10%) to transition to fluff phase.
2. **Fluff phase:** Transaction is broadcast normally to all peers
   using standard INV/TX relay.

### Stem Routing

Each node maintains a stem relay map, refreshed every ~10 minutes:
- For each incoming stem transaction, forward to the assigned stem peer
- If the stem peer disconnects, immediately fluff the transaction
- Transactions in stem phase are not added to the mempool

### Failsafe

If a stem transaction is not seen back via fluff within a timeout
(~30 seconds), the node fluffs it directly. This prevents
transactions from being "black-holed" by offline stem peers.

### Integration with Erlay (IIP-0204)

Dandelion++ operates on the initial relay of a transaction. Once
in fluff phase, Erlay reconciliation applies normally for efficient
propagation.
