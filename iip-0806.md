# IIP-806: Lightning Multi-Path Payments

## Abstract

This IIP specifies multi-path payments (MPP) for INTcoin Lightning,
enabling a single payment to be split across multiple routes for
improved success probability and privacy.

## Motivation

Single-path payments fail if any channel along the route has
insufficient liquidity. MPP splits the payment into smaller parts
routed independently, increasing the probability that each part
succeeds and improving privacy by obscuring the total payment amount
from any single intermediary.

## Specification

### Payment Splitting

The sender splits the total amount into parts, each with:
- Unique `payment_secret` shared across all parts
- Same `payment_hash` (recipient reconstructs full payment)
- Independent route per part (IIP-0803 onion routing)

### Recipient Reconstruction

The recipient holds all partial HTLCs until the total equals the
invoice amount, then settles all parts atomically by revealing the
preimage.

### Minimum Part Size

Minimum part size: 1 INT (10,000 base units). This prevents
excessive splitting that would increase routing fees.
