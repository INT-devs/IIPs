# IIP-102: Replace-By-Fee Signaling

## Abstract

This IIP specifies opt-in Replace-By-Fee (RBF) for INTcoin,
following the signaling mechanism defined in BIP-125. A transaction
signals RBF replaceability by setting any input's nSequence to a
value less than `0xfffffffe`. Miners and mempools may replace such
transactions with higher-fee alternatives.

## Motivation

Users occasionally need to increase the fee on an unconfirmed
transaction — for example when network congestion increases after
broadcast. RBF provides a standard mechanism for this without
requiring recipient cooperation.

## Specification

### Signaling

A transaction is considered RBF-replaceable if **any** of its inputs
has `nSequence < 0xfffffffe` (4294967294).

| nSequence value | RBF signal | Locktime signal |
|-----------------|-----------|-----------------|
| < 0xfffffffe | Replaceable | May also encode relative locktime (IIP-0104) |
| 0xfffffffe | Not replaceable | Locktime enabled, no RBF |
| 0xffffffff | Not replaceable | Locktime disabled |

This is identical to BIP-125.

### Replacement rules

A replacement transaction must satisfy all of the following:

1. **Spends at least one of the same inputs** as the original.
2. **Pays a higher absolute fee** than the sum of all transactions
   it displaces (the original plus any descendants).
3. **Pays a higher fee rate** (INT/vB) than the original.
4. **Does not evict more than 100 transactions** from the mempool
   (the original plus descendants).
5. **Does not introduce new unconfirmed inputs** that the original
   did not already spend.

### Mempool behaviour

When a valid replacement arrives:

1. The original transaction and all its in-mempool descendants are
   removed.
2. The replacement is added.
3. The replacement is relayed to peers.

Nodes must not relay a replacement that fails any of the five rules
above.

### Wallet integration

The INTcoin wallet supports RBF via:

- `bumpfee <txid> [options]` — creates and broadcasts a replacement
  with a higher fee.
- `psbtbumpfee <txid> [options]` — creates an unsigned PST
  (IIP-0404) for external signing.

The Qt wallet exposes RBF through the transaction history context
menu for any unconfirmed transaction that signals replaceability.

### Detection

```cpp
bool bip125_replaceable = false;
for (const auto& vin : tx.vin) {
    if (vin.nSequence < 0xfffffffe) {
        bip125_replaceable = true;
        break;
    }
}
```

## Divergences from BIP-125

None. The signaling mechanism and replacement rules are identical to
BIP-125. The only difference is the underlying signature scheme
(ML-DSA-87 vs ECDSA), which does not affect RBF logic.

## Rationale

Opt-in RBF balances user flexibility (fee bumping) with recipient
expectations (non-replaceable by default). The five replacement
rules prevent abuse: rule 2 ensures miners are compensated, rule 4
prevents mempool churn attacks, and rule 5 prevents pinning attacks.

## Backwards Compatibility

Not applicable. RBF has been supported since genesis.
