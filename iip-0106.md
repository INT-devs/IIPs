# IIP-106: OP_CHECKLOCKTIMEVERIFY

## Abstract

This IIP documents INTcoin's adoption of BIP-65, providing absolute
timelock enforcement at the script level via OP_CHECKLOCKTIMEVERIFY
(CLTV). The mechanism is implemented without modification and has
been active since genesis.

## Specification

Implements [BIP-65](https://github.com/bitcoin/bips/blob/master/bip-0065.mediawiki)
without divergence.

Opcode `0xb1` (redefines OP_NOP2). Verifies that the top stack item
is less than or equal to the transaction's nLockTime and that the
nLockTime type (block height or Unix timestamp) matches.

Fails the script if:
- The stack is empty.
- The top stack item is negative.
- The lock time type (height vs time) does not match nLockTime.
- The top stack item exceeds nLockTime.
- The input's nSequence is `0xffffffff` (locktime disabled).

## Reference Implementation

`src/script/interpreter.cpp` (opcode handler),
`src/script/interpreter.h` (`SCRIPT_VERIFY_CHECKLOCKTIMEVERIFY` flag)
