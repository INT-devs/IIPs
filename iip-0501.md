# IIP-501: Block Explorer REST API

## Abstract

This IIP specifies the public REST API for the INTcoin block explorer,
providing a standardised interface for querying blockchain data.
The API enables third-party applications, wallets, and services to
access block, transaction, address, and mempool data without running
a full node.

## Motivation

Not all applications can run a full node. A standardised explorer
API allows lightweight clients, mobile wallets, web applications,
and exchange integrations to query blockchain state via HTTP.

## Specification

### Base URL

```
https://explorer.international-coin.org/api/v1
```

### Endpoints

#### Blockchain

| Method | Path | Description |
|--------|------|-------------|
| GET | `/info` | Chain info (height, hash, difficulty) |
| GET | `/block/:hash` | Block by hash |
| GET | `/block/height/:n` | Block by height |
| GET | `/block/latest` | Latest block |
| GET | `/blocks?offset=&limit=` | Paginated block list |

#### Transactions

| Method | Path | Description |
|--------|------|-------------|
| GET | `/tx/:txid` | Transaction by txid |
| GET | `/tx/:txid/hex` | Raw transaction hex |
| POST | `/tx/send` | Broadcast raw transaction |

#### Addresses

| Method | Path | Description |
|--------|------|-------------|
| GET | `/address/:addr` | Address summary (balance, tx count) |
| GET | `/address/:addr/txs?offset=&limit=` | Address transactions |
| GET | `/address/:addr/utxos` | Unspent outputs for address |

#### Mempool

| Method | Path | Description |
|--------|------|-------------|
| GET | `/mempool` | Mempool summary (size, fees) |
| GET | `/mempool/txids` | Transaction IDs in mempool |
| GET | `/mempool/recent` | Recent mempool entries |

#### Fee Estimation

| Method | Path | Description |
|--------|------|-------------|
| GET | `/fee/estimates` | Fee estimates by confirmation target |

### Response Format

All responses are JSON with consistent structure:

```json
{
  "data": { ... },
  "meta": {
    "timestamp": 1776368870,
    "height": 12345
  }
}
```

Error responses:

```json
{
  "error": {
    "code": 404,
    "message": "Transaction not found"
  }
}
```

### Rate Limiting

| Tier | Requests/min | Notes |
|------|-------------|-------|
| Public | 60 | No API key required |
| Registered | 300 | Free API key |
| Premium | 3000 | Paid tier |

Rate limit headers: `X-RateLimit-Limit`, `X-RateLimit-Remaining`,
`X-RateLimit-Reset`.

### WebSocket

Real-time updates via WebSocket at `wss://explorer.international-coin.org/ws`:

| Event | Description |
|-------|-------------|
| `block` | New block (hash, height, txcount) |
| `tx` | New mempool transaction |
| `addr:<address>` | Activity on subscribed address |

### CORS

The API supports CORS for browser-based applications. All origins
are allowed for GET requests. POST requests require an API key.

## Security Considerations

- Public API — no sensitive data exposed
- Rate limiting prevents abuse
- POST `/tx/send` validates transactions before relay
- HTTPS required for all connections

## Reference Implementation

- `src/explorer/` — Built-in HTTP/WebSocket explorer server (port 2280)
- Public instance: `https://explorer.international-coin.org`
