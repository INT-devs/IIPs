# IIP-304: Post-Quantum Signature Aggregation

## Abstract

This IIP documents the research direction for post-quantum signature
aggregation in INTcoin, aiming to reduce the on-chain footprint of
Dilithium5 signatures through aggregation techniques.

## Motivation

Dilithium5 signatures are 4,627 bytes each — a major contributor to
transaction and block size. A multi-input transaction with N inputs
carries N * 4,627 bytes of signature data. Signature aggregation
would replace N individual signatures with a single aggregate
signature, dramatically reducing transaction size.

## Research Directions

### Cross-Input Aggregation

Aggregate all signatures within a single transaction into one:
- Reduces N * 4,627 bytes to ~4,627 bytes per transaction
- Requires interactive signing for multi-party transactions

### Half-Aggregation

Non-interactive half-aggregation reduces signature size by ~50%:
- N signatures compressed to ~(N/2 + 1) * 4,627 bytes
- No interaction required between signers

### Lattice-Based Aggregate Signatures

Research areas:
- **Compact aggregate signatures from lattices** (Boneh, Kim)
- **Sequential aggregate signatures** (verifier can add to aggregate)
- **NIST standards track:** Waiting for NIST to standardise
  aggregate signature schemes in future PQC rounds

### Timeline

Dependent on academic research maturity. Current lattice-based
aggregate signature schemes are not yet practical for deployment.
Expected readiness: 2029-2030.

## Impact Estimate

| Scenario | Current Size | With Aggregation |
|----------|-------------|------------------|
| 2-input tx | ~9,400 B sigs | ~4,700 B |
| 5-input tx | ~23,200 B sigs | ~4,700 B |
| 100-tx block | ~930 KB sigs | ~470 KB |

## Reference

This is an informational IIP tracking research progress. No
implementation until a scheme is proven secure and practical.
