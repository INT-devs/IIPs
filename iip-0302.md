# IIP-302: Confidential Transactions

## Abstract

This IIP specifies Confidential Transactions (CT) for INTcoin,
hiding transaction amounts using Pedersen commitments while
preserving the ability to verify that no inflation has occurred.

## Motivation

Standard transactions reveal exact amounts to all observers.
Confidential Transactions hide the amount in a cryptographic
commitment. Validators can verify that inputs equal outputs
(no inflation) without knowing the actual values.

## Specification

### Pedersen Commitments

Each output amount is replaced by a commitment:
```
C = r*G + v*H
```
Where `r` is a blinding factor, `v` is the value, and `G`, `H`
are generator points.

### Range Proofs

Each commitment includes a zero-knowledge range proof demonstrating
the committed value is non-negative and within a valid range
(0 to MAX_MONEY). This prevents negative-value outputs that would
create inflation.

### Post-Quantum Considerations

Standard Pedersen commitments rely on the discrete logarithm
assumption, which is broken by quantum computers. This IIP will
specify a lattice-based or hash-based commitment scheme that
maintains post-quantum security, aligning with INTcoin's security
model.

Research options:
- Lattice-based commitments (Module-LWE)
- Hash-based commitments (statistically hiding)
- Hybrid approach (classical + PQ)

## Status

Research phase. Commitment scheme selection depends on post-quantum
range proof availability and performance characteristics.

## Reference Implementation

Planned for Q1 2028.
