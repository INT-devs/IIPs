# IIP-808: Lightning Channel Splicing

## Abstract

This IIP specifies channel splicing for INTcoin Lightning, enabling
channels to be resized (funds added or removed) without closing and
reopening, maintaining routing availability throughout.

## Motivation

Without splicing, adjusting channel capacity requires closing the
channel (on-chain transaction, timelock delay) and opening a new one
(another on-chain transaction, confirmation wait). Splicing performs
the resize in a single on-chain transaction while the channel remains
operational.

## Specification

### Splice-In

Add funds to an existing channel:
1. Parties negotiate the splice amount
2. New funding transaction created spending the old funding output
   plus additional inputs
3. Channel continues operating with the old funding output until the
   splice transaction confirms
4. After confirmation, channel capacity reflects the new amount

### Splice-Out

Remove funds from a channel (partial withdrawal):
1. Parties negotiate the withdrawal amount and destination
2. New funding transaction created with reduced channel output plus
   a withdrawal output
3. Same transition process as splice-in

### Concurrent Operation

During the splice, the channel operates on both the old and new
funding transactions simultaneously. Commitment transactions are
maintained for both until the splice confirms.
