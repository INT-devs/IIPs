# IIP-204: Erlay Transaction Reconciliation

## Abstract

This IIP documents INTcoin's adoption of BIP-330 Erlay for efficient
transaction relay using set reconciliation. Erlay reduces transaction
announcement bandwidth by ~84% compared to flooding, improving
scalability with increased peer connectivity. The mechanism is
implemented without modification.

## Specification

Implements [BIP-330](https://github.com/bitcoin/bips/blob/master/bip-0330.mediawiki)
without divergence.

### Protocol

Instead of sending INV messages for every transaction to every peer,
Erlay peers periodically reconcile their transaction sets using
Minisketch-based set difference computation.

### Messages

| Message | Description |
|---------|-------------|
| SENDTXRCNCL | Signal reconciliation support during handshake |
| REQTXRCNCL | Request a reconciliation round |
| TXRCNCL | Reconciliation response |
| SKETCH | Minisketch data for set difference |
| REQSKETCHEXT | Request extended sketch (full wtxids for short IDs) |

### Minisketch

When compiled with Minisketch support (`HAVE_MINISKETCH`), uses
proper BCH code-based set reconciliation. Without Minisketch, falls
back to naive encoding (correct but less bandwidth-efficient).

### Benefits

- ~84% bandwidth reduction for transaction relay.
- Better scaling with increased connectivity.
- Reduced fingerprinting through timing analysis (reconciliation
  batches obscure individual transaction timing).
