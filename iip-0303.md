# IIP-303: Zero-Knowledge Range Proofs

## Abstract

This IIP specifies zero-knowledge range proofs for INTcoin,
complementing Confidential Transactions (IIP-0302) by proving
committed values are within valid bounds without revealing the values.

## Motivation

Pedersen commitments (IIP-0302) hide amounts but a malicious user
could commit to a negative value, creating inflation. Range proofs
demonstrate that each committed value lies in [0, MAX_MONEY]
without revealing the value itself.

## Specification

### Proof System Options

| System | Proof Size | Verify Time | PQ-Safe |
|--------|-----------|-------------|---------|
| Bulletproofs | ~700 B | ~3ms | No |
| Bulletproofs+ | ~600 B | ~2.5ms | No |
| Lattice-based | ~10-50 KB | ~10ms | Yes |
| STARK-based | ~50-100 KB | ~5ms | Yes |

### Post-Quantum Selection

INTcoin requires post-quantum range proofs. The selection will be
made based on proof size, verification performance, and maturity
of the scheme. Leading candidates:

1. **Lattice-based range proofs** — Compact but newer research
2. **STARK-based proofs** — Larger but well-understood PQ security
3. **Hash-based commitments with cut-and-choose** — Simple but large

### Aggregation

Multiple range proofs in a single transaction can be aggregated
to reduce total proof size. Aggregated proofs for N outputs are
significantly smaller than N individual proofs.

## Status

Research phase. Depends on maturity of post-quantum ZK proof systems.

Planned for Q1 2028.
