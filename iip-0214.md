# IIP-214: CJDNS Transport

## Abstract

This IIP specifies how INTcoin nodes connect over the CJDNS encrypted
IPv6 mesh network. CJDNS provides end-to-end encryption and
self-certifying addresses using the fc00::/8 IPv6 range.

## Motivation

CJDNS offers a third privacy network option alongside Tor (IIP-0212)
and I2P (IIP-0213). Its design focuses on encrypted mesh networking
rather than anonymity — addresses are derived from public keys, so
connections are authenticated but routable identity is visible within
the mesh. This is useful for private network deployments where
participants trust each other but want encrypted, censorship-resistant
connectivity.

## Specification

### Address Format

CJDNS addresses occupy the fc00::/8 IPv6 range. Each address is
derived from the double-SHA-512 of the node's public key, truncated
to 16 bytes with the first byte forced to `0xFC`. Serialised as
standard IPv6 in ADDRv2 (IIP-0202).

### Detection

INTcoin detects CJDNS addresses by checking the first byte:

```cpp
bool IsCJDNSAddress(const CNetAddr& addr) {
    return addr.IsIPv6() && addr.GetByte(15) == 0xFC;
}
```

### Connection Flow

CJDNS uses standard IPv6 sockets (AF_INET6). No special proxy or
tunnel is required — the operating system routes fc00::/8 traffic
through the CJDNS virtual interface.

1. Node detects CJDNS-capable interface (fc00::/8 bound)
2. Standard TCP connect to peer's fc00:: address on P2P port
3. IIP-0200 V2 handshake proceeds as normal
4. CJDNS address advertised to peers via ADDRv2

### Configuration

```ini
# intcoin.conf
cjdns=1                        # Enable CJDNS support
onlynet=cjdns                  # Optional: CJDNS-only mode
```

### Network Grouping

For eclipse attack resistance (IIP-0200 diversity requirements),
all CJDNS addresses are treated as a single network group since
there is no meaningful IP prefix grouping in the fc00::/8 space.

### Privacy Considerations

- CJDNS provides encryption but not anonymity — addresses are
  tied to public keys
- Within the mesh, routing paths are visible to intermediate nodes
- CJDNS is best suited for trusted mesh deployments, not anonymous
  operation
- All connections still use IIP-0200 V2 post-quantum encryption

## Compatibility

CJDNS addresses are standard IPv6 and work with ADDRv2 (IIP-0202).
Nodes without CJDNS routing simply cannot reach fc00::/8 addresses.
