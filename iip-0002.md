# IIP-2: Community Governance

## Abstract

This IIP establishes the governance framework for INTcoin protocol
changes, including consensus rule modifications, soft fork activation,
and community decision-making processes.

## Motivation

As INTcoin matures, protocol changes must reflect community consensus
rather than unilateral developer decisions. A clear governance process
ensures changes are properly discussed, evaluated, and activated with
broad support.

## Specification

### Decision Categories

| Category | Process | Threshold |
|----------|---------|-----------|
| Consensus changes (hard/soft fork) | IIP + miner signaling (IIP-0104) | 90% miner activation |
| Protocol extensions (P2P, RPC) | IIP + developer review | Rough consensus |
| Process changes (IIP workflow) | IIP + community discussion | Rough consensus |
| Emergency security fixes | Core team discretion | Post-hoc disclosure |

### IIP Lifecycle for Consensus Changes

1. **Discussion:** Proposal in IIP Discussion forum
2. **Draft IIP:** Formal specification submitted as pull request
3. **Review:** 90-day review period with community feedback
4. **Testnet deployment:** Implementation on testnet/signet
5. **Signaling:** IIP-0104 version bits miner signaling
6. **Activation:** Lock-in after 90% threshold over 2016 blocks
7. **Final:** IIP status updated after successful activation

### Community Discussion

INTcoin is a proof-of-work network with no on-chain governance.
Non-consensus decisions are resolved through off-chain discussion:

- **IIP Discussion forum:** https://github.com/INT-devs/IIPs/discussions/
- **Discord #development channel**
- **Mailing list / community calls** (when established)

Rough consensus is determined by the core maintainers after
considering community feedback. There is no coin-weighted voting
or on-chain governance mechanism — INTcoin follows the Bitcoin
tradition of social consensus backed by miner signaling for
consensus rule changes only.

### Emergency Process

Security vulnerabilities bypass normal governance:
1. Responsible disclosure to security@international-coin.org
2. Core team develops and tests fix privately
3. Coordinated disclosure with patch release
4. Post-incident IIP documenting the vulnerability and fix
