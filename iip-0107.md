# IIP-107: Relative Lock-Time Using nSequence

## Abstract

This IIP documents INTcoin's adoption of BIP-68, repurposing the
transaction input nSequence field to encode relative lock-times. The
mechanism is implemented without modification and has been active
since genesis.

## Specification

Implements [BIP-68](https://github.com/bitcoin/bips/blob/master/bip-0068.mediawiki)
without divergence.

When bit 31 of nSequence is clear, the field encodes a relative
lock-time. Bit 22 selects the unit:

| Bit 22 | Unit | Granularity |
|--------|------|-------------|
| 0 | Block height | 1 block |
| 1 | Elapsed time | 512 seconds |

The lower 16 bits encode the lock-time value. The input may not be
included in a block until the specified number of blocks or time
units have elapsed since the output it spends was confirmed.
