# IIP-807: Lightning Trampoline Routing

## Abstract

This IIP specifies trampoline routing for INTcoin Lightning, enabling
lightweight nodes (mobile wallets) to make payments without storing
the full network graph by delegating pathfinding to trampoline nodes.

## Motivation

INTcoin Lightning gossip messages are ~30x larger than Bitcoin
Lightning due to Dilithium5 signatures (IIP-0804). Storing the full
network graph on a mobile device is impractical. Trampoline routing
delegates pathfinding to well-connected nodes while maintaining
payment privacy through nested onion routing.

## Specification

### Trampoline Onion

The sender constructs a two-layer onion:
1. **Outer onion:** Routes to the trampoline node
2. **Inner onion:** Instructs the trampoline to forward to the
   final destination (or next trampoline)

The trampoline node finds the route for the inner portion using its
full graph knowledge, without learning the sender's identity.

### Multiple Trampolines

For additional privacy, the sender can chain multiple trampolines.
Each trampoline only knows its predecessor and the next trampoline,
not the full payment path.
