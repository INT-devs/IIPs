# IIP-203: Compact Block Filters

## Abstract

This IIP documents INTcoin's adoption of BIP-157 and BIP-158 compact
block filters for light client support. Golomb-Coded Set (GCS)
filters allow light clients to efficiently determine whether a block
may contain relevant transactions without downloading the full block.
The mechanism is implemented without modification.

## Specification

Implements [BIP-157](https://github.com/bitcoin/bips/blob/master/bip-0157.mediawiki)
(client protocol) and
[BIP-158](https://github.com/bitcoin/bips/blob/master/bip-0158.mediawiki)
(filter construction) without divergence.

### Filter type

| Type | Name | Contents |
|------|------|----------|
| 0x00 | Basic | All scriptPubKeys of outputs created or spent in the block |

### Messages

| Message | Description |
|---------|-------------|
| GETCFILTERS | Request filters for a range of blocks |
| CFILTER | A single block filter |
| GETCFHEADERS | Request filter headers for a range |
| CFHEADERS | Filter headers (for chain verification) |
| GETCFCHECKPT | Request filter header checkpoints |
| CFCHECKPT | Evenly-spaced filter header checkpoints |

### Index

The node maintains a persistent block filter index. Filters are built
during block connection and stored alongside the block data. The
index tracks the best synced height and filter count.

### Light client workflow

1. Download filter headers (small, verifiable chain).
2. Download filters for blocks of interest.
3. Match wallet scripts against each filter.
4. Download full block only if filter matches.
