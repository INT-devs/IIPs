# IIP-403: Wallet Labels

## Abstract

This IIP specifies the wallet label export/import format for INTcoin,
following the BIP-329 JSON Lines standard. Labels for addresses and
transactions can be exported, backed up, and transferred between
wallets.

## Motivation

Users annotate addresses ("cold storage", "exchange deposit") and
transactions ("payment for invoice #123") for bookkeeping. A standard
portable format allows these labels to survive wallet migrations and
be shared across devices.

## Specification

### Format

Labels are stored as JSON Lines (one JSON object per line) with the
following structure:

**Address label:**
```json
{"type": "addr", "ref": "int1q...", "label": "Cold storage"}
```

**Transaction label:**
```json
{"type": "tx", "ref": "abc123...", "label": "Payment for invoice"}
```

| Field | Type | Description |
|-------|------|-------------|
| type | string | `"addr"` or `"tx"` |
| ref | string | Address (Bech32, IIP-0400) or txid (hex) |
| label | string | User-assigned label text |

### Storage

Transaction labels are stored in the wallet database with key format
`"txlabel" + txid` (32-byte txid). Address labels are stored
separately from transaction labels per BIP-329.

### RPC interface

| Command | Description |
|---------|-------------|
| `exportlabels [filename]` | Export all labels to JSONL file |
| `importlabels <filename>` | Import labels from JSONL file |
| `setlabel <address> <label>` | Set label for an address |
| `getaddressesbylabel <label>` | Get addresses with a specific label |
| `listlabels` | List all wallet labels |

### Divergences from BIP-329

None in the format itself. The only difference is that address
references use INTcoin's Bech32 prefixes (`int1q`, `tint1q`,
`sint1q`) rather than Bitcoin's (`bc1`, `tb1`).

## Rationale

JSON Lines was chosen for streaming compatibility — labels can be
appended, tailed, and processed line-by-line without loading the
entire file. This matches BIP-329's design and enables interop with
standard Unix tooling.

## Backwards Compatibility

Not applicable. Label support has been present since genesis.
