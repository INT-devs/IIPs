# IIP-405: Reusable Payment Codes

## Abstract

This IIP will specify reusable payment codes for INTcoin, adapted
from BIP-47. A payment code is a static identifier that can be
publicly shared. Senders use the recipient's payment code to derive
unique one-time addresses for each payment, providing address privacy
without requiring interactive communication for each transaction.

## Motivation

Sharing a single address for multiple payments links all transactions
to one identity. Generating a new address per payment requires
out-of-band communication. Payment codes solve this by allowing
anyone with the code to independently derive fresh addresses for the
recipient.

## Status

**Draft — not yet implemented.** This IIP is a placeholder for the
privacy roadmap. The specification will be completed when development
begins.

## Specification (preliminary)

The BIP-47 payment code model will be adapted with the following
anticipated divergences:

- **Key exchange:** BIP-47 uses ECDH (Diffie-Hellman on secp256k1)
  to derive shared secrets between sender and recipient. INTcoin will
  use a post-quantum key exchange mechanism (likely ML-KEM-1024 or a
  hash-based shared-secret derivation from Dilithium5 public keys).
  The exact construction is TBD.

- **Notification transaction:** BIP-47 v1 uses an on-chain
  notification transaction to establish the payment channel. This
  mechanism will be evaluated for INTcoin — the larger key sizes
  (2,592-byte public keys) may require an alternative notification
  approach using OP_RETURN with extended capacity (IIP-47).

- **Address derivation:** One-time addresses will be derived using
  SHA3-256 instead of HMAC-SHA512, consistent with the INTcoin HD
  derivation scheme (IIP-0301).

- **Payment code format:** The encoded payment code will use a
  Bech32 format with an INTcoin-specific human-readable prefix
  rather than Base58Check.

### RPC commands (planned)

| Command | Description |
|---------|-------------|
| `createpaymentcode` | Generate a new reusable payment code |
| `getpaymentcode` | Get the wallet's payment code |
| `decodepaymentcode` | Decode and validate a payment code |
| `getpaymentcodeaddress <code> [index]` | Derive address from payment code |
| `listpaymentcodenotifications` | List received notification transactions |

### Dependencies

- IIP-0301: HD Key Derivation (address derivation chain)
- IIP-0300: ML-DSA-87 Digital Signature Scheme
- IIP-47: Extended OP_RETURN (notification transactions)
