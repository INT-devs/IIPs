# IIP-809: Lightning Dual-Funded Channels

## Abstract

This IIP specifies dual-funded channels for INTcoin Lightning, where
both parties contribute funds to the channel opening transaction.

## Motivation

Single-funded channels (IIP-0801) require the opener to provide all
initial liquidity. The counterparty starts with zero sending capacity.
Dual-funding allows both parties to commit funds, creating balanced
channels from the start — essential for routing nodes and LSPs.

## Specification

### Interactive Transaction Construction

1. Both parties exchange `tx_add_input` and `tx_add_output` messages
   to collaboratively build the funding transaction
2. Each party adds their inputs and a change output
3. Both sign the commitment transactions before signing the
   funding transaction
4. Funding transaction is broadcast

### Liquidity Ads

Nodes can advertise willingness to provide liquidity via gossip:
- Available amount
- Fee rate (base + proportional)
- Lease duration

This creates a marketplace for inbound liquidity.
