# IIP-417: Ethereum Bridge (Wrapped INT)

## Abstract

This IIP specifies a bridge protocol for creating ERC-20 wrapped
INT (wINT) tokens on Ethereum, enabling INTcoin to participate in
the Ethereum DeFi ecosystem.

## Motivation

Ethereum's DeFi ecosystem provides lending, DEX trading, and yield
opportunities. A bridge allows INTcoin holders to move INT to
Ethereum as an ERC-20 token and back, expanding utility without
modifying INTcoin's base layer.

## Specification

### Bridge Architecture

**Lock-and-Mint / Burn-and-Release:**

1. User sends INT to a bridge multisig address on INTcoin
2. Bridge validators confirm the deposit (N-of-M threshold)
3. wINT ERC-20 tokens are minted on Ethereum
4. To redeem: user burns wINT on Ethereum
5. Bridge validators release INT from the multisig

### Bridge Validators

A federation of validators (initially project-operated, later
decentralised) monitors both chains:
- Minimum 5 validators, 3-of-5 threshold
- Each validator runs an INTcoin full node and Ethereum light client
- Validators sign attestations using Dilithium5 (INTcoin side)
  and ECDSA (Ethereum side)

### wINT Token

- ERC-20 compliant
- 4 decimal places (matching COIN=10000)
- Mint/burn controlled by bridge contract
- 1:1 peg with INT

### Fee Structure

- Bridge fee: 0.1% of transferred amount
- Ethereum gas fees paid by the user

## Security Considerations

- Bridge multisig is the trust bottleneck
- Regular proof-of-reserves attestation
- Emergency pause mechanism if anomaly detected
- Future: replace federation with ZK proofs of INTcoin state
