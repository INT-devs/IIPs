# IIP-108: OP_CHECKSEQUENCEVERIFY

## Abstract

This IIP documents INTcoin's adoption of BIP-112, providing relative
timelock enforcement at the script level via OP_CHECKSEQUENCEVERIFY
(CSV). The mechanism is implemented without modification and has been
active since genesis.

## Specification

Implements [BIP-112](https://github.com/bitcoin/bips/blob/master/bip-0112.mediawiki)
without divergence.

Opcode `0xb2` (redefines OP_NOP3). Verifies that the top stack item
is satisfied by the input's nSequence value, using the relative
lock-time semantics defined in IIP-0107 (BIP-68).

Fails the script if:
- The stack is empty.
- The top stack item is negative.
- The disable flag (bit 31) is clear in the stack item but set in
  nSequence (lock-time demanded but disabled on input).
- The lock-time type (height vs time) does not match.
- The required relative lock-time exceeds the input's nSequence
  lock-time.
