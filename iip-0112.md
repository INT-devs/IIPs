# IIP-112: AssumeUTXO Snapshots

## Abstract

This IIP specifies the AssumeUTXO mechanism for fast initial sync
by loading a pre-validated UTXO set snapshot. New nodes can begin
validating new blocks immediately while background-validating the
full historical chain.

## Motivation

Full IBD requires downloading and validating every block from genesis.
As the chain grows, this takes longer. AssumeUTXO allows a node to
start with a trusted UTXO snapshot at a specific height, becoming
operational in minutes rather than hours.

## Specification

### Snapshot Format

A UTXO snapshot contains:
- Block hash and height of the snapshot point
- SHA3-256 hash of the serialised UTXO set
- The complete UTXO set (all unspent outputs)

### Hardcoded Snapshots

Snapshot hashes are hardcoded in the source code after community
verification. Each release includes one or more snapshot heights.

### Sync Process

1. Node loads snapshot UTXO set and verifies hash
2. Node begins validating new blocks from the snapshot height
3. Background thread validates historical blocks from genesis
4. When background validation reaches the snapshot height, the
   two UTXO sets are compared — mismatch triggers shutdown

### RPC Commands

- `dumptxoutset <path>` — Dump current UTXO set for snapshot creation
- `loadtxoutset <path>` — Load UTXO snapshot for fast sync

## Security Considerations

- Snapshot hash is consensus-critical — incorrect hash = chain split
- Background validation ensures eventual full verification
- Operators can opt out by syncing from genesis (default without snapshot)
