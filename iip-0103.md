# IIP-103: Monetary Policy and Consensus Constants

## Abstract

This IIP defines the monetary policy and consensus-critical constants
for INTcoin: the base unit, maximum supply, block reward schedule,
halving interval, block time target, and block size limits.

## Motivation

Immutable monetary parameters must be specified unambiguously so that
all implementations produce identical consensus behaviour. This
document is the normative reference for these constants.

## Specification

### Base unit

| Constant | Value |
|----------|-------|
| COIN | 10,000 base units |
| Decimal places | 4 |
| Display | `1.0000 INT` = 10,000 base units |

One INT is divisible to four decimal places. This is fewer than
Bitcoin's eight but sufficient for the intended supply scale.

### Supply

| Constant | Value |
|----------|-------|
| Initial block reward | 105,113,636 INT (1,051,136,360,000 base units) |
| Halving interval | 1,051,200 blocks (~4 years at 2-minute blocks) |
| Maximum supply | 221,000,000,000,000 INT (2.21 × 10^14) |
| MAX_MONEY | 2,210,000,000,000,000,000 base units |

MAX_MONEY fits in a signed 64-bit integer
(2.21 × 10^18 < 9.22 × 10^18 = INT64_MAX).

The block reward halves every 1,051,200 blocks until it reaches zero.
Halvings occur at blocks 1,051,200; 2,102,400; 3,153,600; and so on.

### Block timing

| Constant | Value |
|----------|-------|
| nPowTargetSpacing | 120 seconds (2 minutes) |
| Blocks per day | 720 |
| Blocks per year | ~262,800 |

### Difficulty adjustment (Digishield V5)

Difficulty adjusts every block using the Digishield V5 algorithm.

| Parameter | Value |
|-----------|-------|
| Averaging window | 17 blocks |
| Target timespan | 2,040 seconds (17 × 120s) |
| Max adjustment down | 32% per block |
| Max adjustment up | 16% per block |
| fPowAllowMinDifficultyBlocks | false (mainnet), true (testnet/regtest) |

### Block limits

| Limit | Value |
|-------|-------|
| MAX_BLOCK_SIZE | 4,000,000 bytes (4 MB) |
| MAX_BLOCK_WEIGHT | 16,000,000 weight units |
| MAX_BLOCK_SIGOPS | 80,000 |
| Max transactions per block | 10,000 |
| WITNESS_SCALE_FACTOR | 4 |

Weight is computed as `size × WITNESS_SCALE_FACTOR` (no witness
discount in the current protocol; the weight system exists for future
extensibility).

### Coinbase maturity

Coinbase outputs require 100 confirmations before they may be spent.

## Rationale

The 4-decimal-place resolution and large supply are chosen to make
INTcoin suitable for both micro-transactions and large-value
transfers without requiring sub-unit notation. The 2-minute block
time provides faster confirmations than Bitcoin's 10 minutes while
maintaining adequate security margin for RandomX proof-of-work
propagation.

Digishield V5 per-block adjustment with asymmetric damping (16% up,
32% down) prevents the difficulty oscillation and timestamp
manipulation attacks that affect less responsive adjustment
algorithms.

## Backwards Compatibility

Not applicable. These constants have been in effect since genesis.
