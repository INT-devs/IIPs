# IIP-113: Atomic Swap Protocol

## Abstract

This IIP specifies the atomic swap protocol for trustless cross-chain
exchange between INTcoin and other cryptocurrencies using
Hash Time-Locked Contracts (HTLCs).

## Motivation

Atomic swaps enable decentralised exchange without a trusted
intermediary. Users can trade INTcoin for Bitcoin, Litecoin, or other
HTLC-capable cryptocurrencies directly on-chain.

## Specification

### HTLC Construction

The swap uses two HTLCs — one on each chain — locked by the same
SHA3-256 hash. The initiator reveals the preimage to claim their
funds, which simultaneously enables the counterparty to claim theirs.

### Protocol Flow

1. **Alice** (has INT, wants BTC) generates random preimage `R`,
   computes `H = SHA3-256(R)`
2. **Alice** creates HTLC on INTcoin: pays Bob if he reveals `R`
   before timelock T1, else refundable to Alice after T1
3. **Bob** verifies Alice's HTLC, creates matching HTLC on Bitcoin:
   pays Alice if she reveals `R` before timelock T2 (T2 < T1)
4. **Alice** claims Bob's BTC HTLC by revealing `R`
5. **Bob** uses revealed `R` to claim Alice's INT HTLC

### Timelock Safety

T1 (INTcoin) must be significantly longer than T2 (counterparty
chain) to account for block time differences and confirmation
requirements. Recommended: T1 = 48 hours, T2 = 24 hours.

### Script

INTcoin HTLC script using OP_CHECKSIG_DILITHIUM (IIP-0100):

```
OP_IF
  OP_SHA3_256 <hash> OP_EQUALVERIFY
  <bob_pubkey> OP_CHECKSIG_DILITHIUM
OP_ELSE
  <timelock> OP_CHECKLOCKTIMEVERIFY OP_DROP
  <alice_pubkey> OP_CHECKSIG_DILITHIUM
OP_ENDIF
```

### Cross-Chain Hash Compatibility

INTcoin uses SHA3-256 for payment hashes. For swaps with SHA-256
chains (Bitcoin), both chains must use the same hash function for
the preimage lock. The HTLC script supports both SHA3-256 and
SHA-256 hash verification opcodes.

## Security Considerations

- Timelock asymmetry prevents Alice from claiming both sides
- Preimage revelation is atomic — once revealed, both parties can claim
- Network partitions during swap are handled by timelocks
