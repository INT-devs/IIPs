# IIP-810: Lightning Service Provider Framework

## Abstract

This IIP specifies the Lightning Service Provider (LSP) framework
for INTcoin, standardising how service providers offer liquidity,
channel management, and payment forwarding to end-user wallets.

## Motivation

End users (especially mobile wallets) need inbound liquidity to
receive Lightning payments. LSPs provide this as a service, opening
channels to users, managing liquidity, and handling the complexity
of channel maintenance.

## Specification

### LSP Services

| Service | Description |
|---------|-------------|
| Channel opening | LSP opens a channel to the user on demand |
| Inbound liquidity | LSP provides receiving capacity |
| Just-in-time channels | Channel opened when first payment arrives |
| Channel management | Rebalancing, fee optimization |
| Backup | Encrypted channel state backup |

### LSPS Standards

| Standard | Description |
|----------|-------------|
| LSPS0 | Transport and authentication |
| LSPS1 | Channel request API |
| LSPS2 | Just-in-time channel opening |

### JIT Channels (LSPS2)

1. User's wallet registers with LSP
2. Payer sends Lightning payment to user's node
3. LSP intercepts the HTLC, opens a channel to the user with
   the payment amount as initial push
4. User receives the payment via the new channel

### Authentication

LSPs authenticate clients using Dilithium5 node identity keys.
Communication encrypted via IIP-0200 V2 transport.
