# IIP-114: OP_CHECKTEMPLATEVERIFY (Covenants)

## Abstract

This IIP specifies OP_CHECKTEMPLATEVERIFY (CTV) for INTcoin, enabling
covenants — spending conditions that restrict how outputs can be
spent in future transactions.

## Motivation

Currently, once funds are received, the owner can spend them to any
destination. Covenants add constraints on future spending, enabling:

- **Vaults:** Time-delayed withdrawals with emergency clawback
- **Congestion control:** Pre-committed transaction trees that batch
  payouts efficiently
- **Non-interactive channels:** Payment channel constructions without
  online counterparties
- **DLC (Discreet Log Contracts):** Oracle-based conditional payments

## Specification

### Opcode

`OP_CHECKTEMPLATEVERIFY` (CTV) verifies that the spending
transaction matches a committed template hash. The hash commits to:

- Transaction version
- Locktime
- Input count and sequences
- Output count, amounts, and scriptPubKeys
- Input index being spent (for script reuse across inputs)

### Template Hash

```
template_hash = SHA3-256(
    version || locktime || input_count ||
    SHA3-256(sequences) || output_count ||
    SHA3-256(outputs) || input_index
)
```

### Vault Construction

Using CTV, a vault is constructed as:
1. Deposit to vault address (CTV-locked)
2. CTV template commits to a time-delayed withdrawal transaction
3. Withdrawal transaction has a timelock (e.g., 144 blocks / ~5 hours)
4. During the timelock, an emergency key can sweep funds to cold storage
5. After timelock, funds are released to the intended destination

### Activation

Deployed via IIP-0104 version bits with miner signaling.
