# IIP-202: ADDRv2 Address Format

## Abstract

This IIP documents INTcoin's adoption of BIP-155 ADDRv2 for peer
address relay. ADDRv2 extends the original ADDR message to support
variable-length addresses, enabling Tor v3 (.onion), I2P, CJDNS, and
future network types. The mechanism is implemented without
modification.

## Specification

Implements [BIP-155](https://github.com/bitcoin/bips/blob/master/bip-0155.mediawiki)
without divergence.

### Supported network types

| ID | Network | Address length |
|----|---------|---------------|
| 1 | IPv4 | 4 bytes |
| 2 | IPv6 | 16 bytes |
| 3 | Tor v3 | 32 bytes |
| 4 | I2P | 32 bytes |
| 5 | CJDNS | 16 bytes |

### Messages

| Message | Description |
|---------|-------------|
| SENDADDRV2 | Signals ADDRv2 support during handshake |
| ADDRV2 | Relays peer addresses in variable-length format |

Peers that have exchanged SENDADDRV2 use ADDRV2 instead of the
legacy ADDR message for all subsequent address relay.
