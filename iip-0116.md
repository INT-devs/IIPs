# IIP-116: Dynamic Block Size

## Abstract

This IIP specifies a dynamic block size adjustment mechanism for
INTcoin, allowing the maximum block weight to increase or decrease
based on demand while preventing runaway growth.

## Motivation

A fixed block weight limit creates a hard capacity ceiling. During
demand spikes, transactions compete via fees, pricing out small
users. A dynamic limit allows the network to handle temporary
surges while maintaining long-term sustainability through a
growth penalty mechanism.

## Specification

### Adjustment Algorithm

The maximum block weight adjusts based on a trailing median of
actual block weights:

```
new_limit = median(last_2016_block_weights) * growth_factor
```

With constraints:
- `growth_factor` between 1.0x and 2.0x per adjustment period
- Absolute minimum: 1,000,000 weight units (1MW)
- Absolute maximum: 16,000,000 weight units (16MW)
- Blocks exceeding the limit pay an exponential "penalty fee"
  via reduced coinbase reward

### Penalty Mechanism

Miners producing blocks above the base limit forfeit a portion of
their block reward proportional to the excess:

```
penalty = base_reward * (excess_weight / base_limit)^2
```

This creates economic pressure against unnecessary bloat while
allowing temporary expansion when fees justify it.

### Activation

Deployed via IIP-0104 version bits soft fork. The initial limit
remains at 4,000,000 weight units (4MW) until activation.

## Status

Research phase. Requires economic analysis and simulation before
parameter selection.

## Reference Implementation

Planned for Q3 2028.
